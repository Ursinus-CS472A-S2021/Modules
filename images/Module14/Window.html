Window Functions And Convolution/Multiplication Duality¶

Chris Tralie¶

Forward DFT

$X[k] = \sum_{n=0}^{N-1} x[n] e^{-i 2 \pi \frac{k}{N} n}$¶

Inverse DFT

$x[n] = \frac{1}{N}\sum_{k=0}^{N-1} X[k] e^{i 2 \pi \frac{k}{N} n}$¶

Convolution in Time Domain: Just as the DFT of the convolution of two signals is the product of those signals' DFTs,

Convolution in Frequency Domain: The product of two signals is the convolution of the DFTs of the two signals

def sin_window(N):
    """
    Create the sine window sin(pi*n/N)
    
    Parameters
    ----------
    N: int
        Length of window
    
    Returns
    -------
    ndarray(N): Samples of the window
    """
    return np.sin(np.pi*np.arange(N)/N)

def hann_window(N):
    """
    Create the Hann window 0.5*(1-cos(2pi*n/N))
    
    Parameters
    ----------
    N: int
        Length of window
    
    Returns
    -------
    ndarray(N): Samples of the window
    """
    return 0.5*(1 - np.cos(2*np.pi*np.arange(N)/N))


def blackman_harris_window(N):
    """
    Create a Blackman-Harris Window
    
    Parameters
    ----------
    N: int
        Length of window
    
    Returns
    -------
    ndarray(N): Samples of the window
    """
    a0 = 0.35875
    a1 = 0.48829
    a2 = 0.14128
    a3 = 0.01168
    t = np.arange(N)/N
    return a0 - a1*np.cos(2*np.pi*t) + a2*np.cos(4*np.pi*t) - a3*np.cos(6*np.pi*t)

Square Wave with Square Window¶

The default window function is a "square window," where we don't actually do anything to the signal after we window it; we simply chop it off at the ends with no tapering. This introduces many spurious "side lobes" (wiggles), which we saw before as frequency bleeding, but which we now understand as the convolution of the frequency response of the square window with the frequency response of the windowed signal
